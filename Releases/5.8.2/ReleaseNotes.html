FileNet Connector for Salesforce Release Notes


New Features

The 5.8.2 version of the IBM FileNet Salesforce Connector app introduced the following new features:

  • Avoid the limit of 55 lookup fields in a Salesforce Object Query Language (SOQL) query:
    • Salesforce has implemented a limit on the maximum number of lookup fields that can be retrieved via a single query of 55. However customers can create more than this number of lookup relationships. To address this problem, we have changed our query logic to count the number of lookup fields in a query, and if it will exceed the limit of 55, then the query will be broken up into multiple queries, in order to retrieve all fields
  • Support Salesforce Connector widget on Salesforce Digital Experience Community Pages:
    • The IBM Document List widget can now be added to any Salesforce Digital Experience Community record pages.
  • The ability to synchronize a Salesforce Picklist field:
    • The Salesforce Connector supports synchronizing fields from a Salesforce record, onto properties of a FileNet document that is created from an attachment to that Salesforce record.
    • The Connector currently supports synchronizing Salesforce fields that are of type String, Integer, Picklist, or Date only.
  • Support Choice field values up to 100 characters long: :
    • The previous version of the Salesforce Connector supports a Choice item value of at most 64 characters in length.
    • This version now supports the values up to 100 characters.

Prerequisites

For Salesforce Connector versions 5.6.3, 5.6.4, 5.6.5, 5.6.6 and 5.7.0:

  • The Salesforce Connector is dependent on the Content Services GraphQL API version 5.5.8 or higher.
  • However for the configurations where support for multiple Salesforce organizations is needed, then version 5.5.8 IF001 or above Content Services GraphQL API is required.
