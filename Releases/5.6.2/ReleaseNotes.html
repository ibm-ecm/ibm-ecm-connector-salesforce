FileNet Connector for Salesforce Release Notes


This release includes the following updates:

Security Update to prevent Cross-Site Request Forgery (CSRF) Attack

IBM FileNet Salesforce Connector app has been updated to support Cross-Site Request Forgery (CSRF) changes implemented in the latest version of IBM FileNet Content Services GraphQL API. An anti-CSRF token has been implemented with the BM FileNet Content Services GraphQL API to prevent a CSRF attack.

Compatibility

The 5.6.2 version of the IBM FileNet Salesforce Connector app depends on the following software components:

  • IBM FileNet Content Services GraphQL API version 5.5.4 IF004 or 5.5.5 IF004 or 5.5.6 IF002
  • IBM FileNet Content Manager version 5.5.4 or later

Known Issues

The IBM FileNet Connector for Salesforce app has the following known issues:

Document file names containing double byte characters are corrupted when downloading documents from the IBM FileNet Salesforce Connector app.

This issue only affects documents whose file names use a double byte character language.  Double byte character languages include Japanese, Chinese, and Korean.  In these languages, the name of the document appears correctly in the app, but if the user downloads the file to their desktop, the name of the file contains corrupted characters.  The contents of the file are unaffected.
